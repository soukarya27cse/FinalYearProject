## 1. What It Does

`dashboard.py` is a read-only analytics frontend for the prediction workbooks
generated by `nifty100_tracker.py`. It does **not** run the model or fetch any
market data — it only reads existing `.xlsx` files and renders them.

On launch it:

1. Scans the `workbooks/` directory for all `.xlsx` files
2. Reads the ticker symbol directly from each workbook (column B, row 3)
3. Populates a dropdown with all discovered companies
4. Loads the selected company's full prediction history
5. Renders KPI cards, interactive charts, a data table, and a multi-ticker
   comparison panel — all updating live when the ticker or date range changes

---

## 2. Project Layout

`dashboard.py` lives at the project root alongside `nifty100_tracker.py`. The
`workbooks/` folder is also at the root, outside `src/`. The dashboard reads
directly from `workbooks/` — no path configuration needed as long as you launch
it from the project root.

```
ticker-teller/                        ← run  streamlit run dashboard.py  here
│
├── dashboard.py                     ← this file — Streamlit web app
├── nifty100_tracker.py              ← generates the workbooks (run first)
├── requirements.txt
│
├── src/                             ← model source (not used by dashboard)
│   ├── __init__.py
│   ├── data.py
│   ├── model.py
│   ├── charts.py
│   └── ui.py
│
├── workbooks/                       ← Excel files read by the dashboard
│   ├── RELIANCE_NS.xlsx             ← one file per company
│   ├── TCS_NS.xlsx
│   ├── HDFCBANK_NS.xlsx
│   └── …  (up to 100 files)
│
├── logs/                            ← tracker logs (not used by dashboard)
└── state/                           ← tracker state  (not used by dashboard)
```

> **Important:** `workbooks/` must exist and contain at least one `.xlsx` file
> before launching the dashboard. Run `python3 nifty100_tracker.py --once` first
> to generate prediction data.

## 5. Usage

### Launch from the project root

```bash
streamlit run dashboard.py
```

Streamlit will print a local URL (usually `http://localhost:8501`). Open it in
your browser.

### Custom workbooks directory

If your workbooks are stored somewhere other than `workbooks/` next to
`dashboard.py`, pass the path via a CLI argument:

```bash
streamlit run dashboard.py -- --workbooks /path/to/your/workbooks
```

> Note the `--` separator between Streamlit's own flags and the app's flags.

### Run on a specific port

```bash
streamlit run dashboard.py --server.port 8080
```

### Run in headless mode (server / remote machine)

```bash
streamlit run dashboard.py --server.headless true --server.port 8501
```

Then open `http://<your-server-ip>:8501` in your browser.

## 6. Dashboard Features

### KPI cards — two rows of live metrics

The first row shows the most important per-ticker numbers:

| Card | Description |
|---|---|
| Last Actual Price | Most recent closing price in the selected date range |
| Last Predicted Price | Most recent MC-Dropout forecast in the range |
| Avg Error (%) | Mean absolute prediction error across all days |
| Median Error (%) | Median absolute error — less sensitive to outliers |
| Directional Accuracy | % of days where the model correctly predicted up vs down |
| Total Return (Actual) | Cumulative % price change from first to last day in range |

The second row shows error distribution details:

| Card | Description |
|---|---|
| Max Error (%) | Worst single-day prediction error |
| Min Error (%) | Best single-day prediction error |
| Std Dev Error | Spread of the error distribution |
| Days Analysed | Number of trading days in the selected range |

All eight cards update instantly when you change the ticker or date range.

### Four chart tabs

See [Section 7](#7-chart-reference) for full details on each chart.

### Data table tab

A formatted table showing every row in the workbook for the selected range.
Features:
- Checkbox to show or hide the verbose feature columns (Price Features,
  Factor Features) which contain the full 22-feature snapshot per day
- **Download as CSV** button — exports the current filtered view to a `.csv`
  file named `<TICKER>_predictions.csv`

### Multi-ticker comparison

A multiselect at the bottom of the page lets you overlay the actual closing
prices of any number of other companies on a single chart. Useful for comparing
sector-level price movements across the 30-day prediction window.

## 7. Chart Reference

### Tab 1 — Price Chart

**Actual Price vs Predicted Price** as two line+marker traces over time.

- **Blue solid line** — actual closing price from Yahoo Finance
- **Pink dashed line** — model's MC-Dropout mean prediction
- **Translucent fill** between the two lines visualises the prediction gap
- Hover shows exact values for both traces on the same date
- X-axis shows dates; Y-axis shows price in ₹

### Tab 2 — Error Chart

**Daily absolute prediction error (%)** as a vertical bar chart.

- **Green bars** — model underestimated the price (Actual > Predicted)
- **Red bars** — model overestimated the price (Actual < Predicted)
- **Yellow dotted line** — average error across the selected range
- The displayed value is always the absolute error; colour encodes direction

### Tab 3 — Scatter Chart

**Actual vs Predicted scatter** — each point is one trading day.

- The **grey dotted diagonal** represents perfect prediction (Actual = Predicted)
- Points **above the diagonal** — model underestimated (green)
- Points **below the diagonal** — model overestimated (red)
- Tighter clustering around the diagonal = better model performance
- Hover shows the exact date, actual price, and predicted price for each point

### Tab 4 — Cumulative Return Chart

**Cumulative % return from the first day** in the selected range.

- **Blue solid line** — actual price normalised to 0% at day 1
- **Purple dashed line** — predicted price normalised to 0% at day 1
- Shows how closely the model tracks the real price trajectory over time
- Requires at least 2 data points; shows an info message otherwise

### Tab 5 — Data Table

Raw data for the selected ticker and date range. Columns:

| Column | Content |
|---|---|
| Date | `YYYY-MM-DD` |
| Ticker | Yahoo Finance symbol |
| Actual Price | Formatted as `₹X,XXX.XX` |
| Predicted Price | Formatted as `₹X,XXX.XX` |
| Error (%) | Absolute error, formatted as `X.XX%` |
| Signed Error (%) | Raw signed error — positive = underestimated, negative = overestimated |
| Price Features *(optional)* | Full 16-feature snapshot string |
| Factor Features *(optional)* | Full 6-factor snapshot string |

## 8. How Data Is Loaded

### Ticker discovery

On startup, `discover_workbooks()` scans `workbooks/` for all `.xlsx` files.
For each file it calls `_peek_ticker()`, which opens the workbook and reads the
actual ticker symbol stored in **column B, row 3** (the first data row). This
ensures that tickers with special characters — `BAJAJ-AUTO.NS`, `M&M.NS` — are
always displayed correctly, regardless of how the filename was sanitised.

If `_peek_ticker()` fails (e.g. an empty or malformed file), it falls back to
reconstructing the ticker from the filename stem.

### Caching

Both `discover_workbooks()` and `load_workbook_data()` are decorated with
`@st.cache_data`. This means:
- Workbooks are scanned once at startup, not on every interaction
- Each workbook is read from disk only once per session; subsequent filter
  changes use the cached DataFrame
- To force a reload (e.g. after the tracker appends new rows), press
  **`R`** to rerun the app or use Streamlit's **Clear Cache** option in the
  top-right menu

### Row parsing

For each data row the dashboard:
1. Reads date, ticker, actual price, predicted price, and feature strings from
   the fixed column positions used by `nifty100_tracker.py`
2. **Recomputes** `Error_pct` and `Signed_Error` from the raw `Actual` and
   `Predicted` values rather than trusting the cached formula result in column G
   (Excel formula values are not always evaluated by `openpyxl` when
   `data_only=True`)
3. Skips any row where `Actual` or `Predicted` cannot be parsed as a float

|---|---|

## 10. Handling Missing or Inconsistent Data

The dashboard is designed to degrade gracefully when data is incomplete,
corrupt, or partially generated.

| Situation | Behaviour |
|---|---|
| `workbooks/` directory does not exist | Sidebar shows an error with instructions to run the tracker first |
| Workbook exists but `Predictions` sheet is missing | File is silently skipped during discovery |
| Data rows have `None` / empty values | Rows are skipped during loading — no crash |
| Formula cells not evaluated by openpyxl | Error % is recomputed from raw Actual and Predicted values |
| `NaN` or `Inf` in numeric columns | Replaced with `None`; affected rows excluded from statistics |
| Date range filter leaves 0 rows | Warning shown; charts and KPIs are not rendered |
| Fewer than 2 rows in range | Cumulative Return tab shows an info message; other charts still render |
| Workbook written by a different tool or version | `_peek_ticker()` falls back to filename-based name; data still loads if column layout matches |

## 11. Configuration

The dashboard has no config file. All tuneable constants are at the top of
`dashboard.py` under clearly labelled comment blocks.

### Column indices

These match the fixed layout written by `nifty100_tracker.py`. Only change
these if you customise the tracker's output format.

```python
_COL_DATE     = 1   # A
_COL_TICKER   = 2   # B
_COL_PRICE_F  = 3   # C  — price branch features
_COL_FACTOR_F = 4   # D  — quant factor features
_COL_ACTUAL   = 5   # E
_COL_PRED     = 6   # F
_COL_ERROR    = 7   # G
_COL_SIGNED   = 8   # H
_DATA_START   = 3   # first data row (rows 1–2 are headers)
```

### Workbook directory resolution order

The dashboard resolves the workbooks path in this priority order:

1. **CLI argument** — `streamlit run dashboard.py -- --workbooks /path`
2. **URL query parameter** — `?workbooks=/path` in the browser address bar
3. **Default** — `workbooks/` folder next to `dashboard.py`
